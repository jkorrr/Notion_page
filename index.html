This project provides a general overview of how different filters and frequencies within images correspond to the ways we perceive them. In this project, we sharpen images, blur images, blend images, and create hybrid images.

1.1: Finite Difference Operator

Two finite difference kernels were created as Numpy arrays for calculating the partial derivatives. The D_x was defined as np.array([[1, -1]]), and the D_y as np.array([[1], [-1]]). These kernels were applied to the original image using scipy’s 2-D convolution function, producing images that represent the partial derivatives in the x and y directions. Then, I found the gradient magnitude by computing np.sqrt(dx_deriv ** 2 + dy_deriv ** 2). This process effectively treats the pixel values from the partial-derivative images as components of the gradient vector and calculates the L2 norm for the final image. 2.1: Image Sharpening

In 2.1, the goal was to sharpen the image. First, we convolve the original image with a gaussian kernel to filter out the higher frequencies in the image by blurring it. Then, the high frequencies are found by subtracting the blurred image by the original image, which removes all the low frequency parts of the image. Then, these high frequencies are emphasized by scaling them with the final sharpened image being final_img = input_img + alpha * high_freq_img as the final result.

2.3: Gaussian & Laplacian Stacks

In 2.3, we have to create two types of stacks a Gaussian and a Laplacian. For the Gaussian stack, what we do is we start with the given image as the first level, and then every subsequent level down, we apply the gaussian filter to the image from the previous level, which makes it seem a little more blended. Then, the Laplacian stack takes the difference between the images at each layer (in more formalized turns, it can be described as: l_stack[i] = g_stack[i] - g_stack[i + 1]. The last layer in the Laplacian stack is the same as the one in the Gaussian, and thus they have the same number of levels.

2.4: Multiresolution Blending

For 2.4, we are given once again two images as input when we are trying to blend them. We have left_img and right_img . We generate the Laplacian stacks for each of these images using 2.3, and then we also have a new mask input image that we apply the Gaussian stack on. In order to blend the images, we apply this formula: final_blended_img = M_g * right_img_stack + (1 - M_g) * left_img_stack (M_g is the mask input image’s Gaussian stack).
